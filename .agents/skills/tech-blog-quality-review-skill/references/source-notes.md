# Source Notes (for review lens design)

以下はレビュー観点を設計する際に参照した公開記事の要点。

## 1. 読者起点と構成
- SocialDog(Zenn): 読者課題を先に置き、結論を早く出す。
- Hatena Developer Blog hints: 読者が必要な情報へ最短で到達できる構成を重視。
- NTT DATA Tech(Zenn): 技術ブログを組織知として残す視点と、再利用性の高い記述を重視。

## 2. 可読性と離脱防止
- ラッコキーワード: 読了率を意識し、文の長さ・改行・見出しで負荷を下げる。
- バズ部(LUCY): Webライティングは「誰に何をどう伝えるか」の明確化が最優先。

## 3. 技術ブログ実務
- Qiita(執筆ノウハウ記事): 経験知を再現可能な形で分解し、読者が実行できる粒度にする。
- Wantedly Tech Blog: 継続発信と品質標準化（レビュー観点の統一）が重要。
- もしゃ修行メディア: 技術記事は体験談だけで終わらせず、再現手順を示す。

## 4. SEO/タイトル設計
- makusan / CROCO などのWeb記事: タイトルは検索意図との一致、本文は結論先出しと見出し最適化が有効。

## 5. 補足
- 一部URLは公開状態や形式の都合で直接参照が安定しない場合があるため、最終判断は一次情報（公式Doc・実測ログ）を優先する。
